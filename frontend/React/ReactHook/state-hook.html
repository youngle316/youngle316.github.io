# React-Hook 之 State Hook

Hook是React在16.8这个版本发布的一个重要的特性。中文含义就是钩子，它可以让你在函数型组件中使用State及React等其他特性。 Hook 是一些可以让你在函数组件里“钩入” React state 及生命周期等特性的函数。Hook 不能在 class 组件中使用 —— 这使得你不使用 class 也能使用 React。

# Hook解决的问题

在之前使用Class编写的组件中，可能会遇到下面的问题。

  • 在组件之中复用状态逻辑很难

虽然React提供了HOC高阶组件和Render Props来实现组件的复用，但是会导致项目的更加复杂，和代码阅读性更差，组件包裹组件又返回组件。使得学习成本急剧上升。

不过可以使用 Hook从组件中提取状态逻辑，使得这些逻辑可以单独测试并复用。**Hook 可以在无需修改组件结构的情况下复用状态逻辑。**就像抽离公共方法一样简单。

  • 复杂组件变的难以理解

在一些复杂的组件中，通常包含着许多的业务逻辑和副作用。组件常常在componentDidMount和componentDidUpdate中获取数据，但是componentDidMount中又会有其他的业务逻辑；有时候还需要考虑组件是否要更新，更新受哪个state或props的影响；包括在componentWillUnmount中去处理取消事件监听等逻辑。这些不同的业务逻辑写在同一个方法中，导致查看某一个逻辑需要跟着生命周期走一遍，会导致一些隐藏的bug。

为了解决这个问题，Hook 将组件中相互关联的部分拆分成更小的函数（比如设置订阅或请求数据），而并非强制按照生命周期划分。

  • 难以理解的class

class中的this比较难理解，在不同的场景下this的指向又是不同的，同时不能忘记绑定事件处理器，通常使用bind或者箭头函数，这也导致了学习成本的增加。

为了解决这些问题，Hook 使你在非 class 的情况下可以使用更多的 React 特性。 从概念上讲，React 组件一直更像是函数。而 Hook 则拥抱了函数，同时也没有牺牲 React 的精神原则。

# 使用State Hook

React官方提供的有关state的Hook有两种：useState和useReducer

# useState

在React源码中，useState是这样定义的

function useState<S>(initialState: S | (() => S)): [S, Dispatch<SetStateAction<S>>];

function useState<S = undefined>(): [S | undefined, Dispatch<SetStateAction<S | undefined>>];

首先，先分别使用class和Hook来实现一个点击按钮，数字加一的案例。

  1. 使用class
import React, {Component} from 'react';

class LikeButtonClass extends Component {
    constructor(props) {
        super(props);
        this.state = {
            like: 0
        }
    }
    render() {
        const {like} = this.state;
        return (
            <div>
                <button onClick={() => this.setState({like: like +1})}>{like} ✌</button>
            </div>
        );
    }
}

export default LikeButtonClass;
  1. 使用Hook
import React, {useState} from 'react';

function LikeButtonHook(props) {
    const [like, setLike] = useState(getInitValue)
    return (
        <div>
            <button onClick={() => setLike(like +1)}>{like} ✌</button>
        </div>
    );
}

export default LikeButtonHook;

上述两个例子中，两种写法是等价的。class那个都很常见就不介绍了。

在使用Hook的写法中，useState就是一个Hook，它可以让我们在非class组件中去使用State这一特性。

const [like, setLike] = useState(0)
  • 声明了一个名为like的state变量，并且默认值为 0
  • 声明了一个名为setLike的方法来改变count的值

需要注意的是，当你的默认值是需要进行计算的时候，可以传一个方法并返回一个默认值

// 不能是一个耗时操作
// const getInitValue = () => {
//     setTimeout(() => {
//         return 2
//     }, 2000)
// }
// 这样的计算可以，返回 1
const getInitValue = () => {
  return 1 > 2 ? 0 : 1
}
const [like, setLike] = useState(getInitValue)

耗时操作需要使用下一个Hook叫做useEffect。

# useReducer

在React源码中，useReducer是这样定义的

function useReducer<R extends ReducerWithoutAction<any>, I>(
  reducer: R,
  initializerArg: I,
  initializer: (arg: I) => ReducerStateWithoutAction<R>
): [ReducerStateWithoutAction<R>, DispatchWithoutAction];

使用的方式如下

const [state, dispatch] = useReducer(reducer, initialArg, init)

参数.

  • reducer是一个函数，接收state和action为参数，返回一个新的state
  • initialArg是初始化state的值
  • init是一个方法，从来惰性初始化state的值

返回值.

返回一个数组

  • state就是定义的state
  • dispatch是一个方法，它的用法与redux中的一样，会传一个action到reducer

下面就是一个简单的计数器的例子

import React, {useReducer} from 'react';

const reducer = (state, action) => {
  switch (action.type) {
    case "decrement":
      return {count: state.count - 1}
    case "increment":
      return {count: state.count + 1}
    case "reset":
      return init(action.payload)
    default :
      throw new Error()
  }
}

const init = (initialCount) => {
  return {count: initialCount};
}

function CountNumHook({initState}) {
  const [state, dispatch] = useReducer(reducer, initState, init)
  return (
    <>
      Count: {state.count}
      <button onClick={() => {dispatch({type: "reset", payload: initState})}}>reset</button>
      <button onClick={() => {dispatch({type: "decrement"})}}>-</button>
      <button onClick={() => {dispatch({type: "increment"})}}>+</button>
    </>
  );
}

export default CountNumHook;

# 两者的异同

  • 相同

都是往函数型组件钩入state特性的Hook

  • 不同
  1. 使用场景不同：通常情况下，useState可以满足绝大部分的场景；但是在state 逻辑较复杂且包含多个子值，或者下一个 state 依赖于之前的 state等，都可以优先使用useReducer

# useState与setState的异同

  • 相同
  1. 作用相同，都可以为更新state
  2. 多个setState或setLike会被合并成为一个
// useState
import React, {useState, useEffect} from 'react';

function LikeButtonHook(props) {
  const [like, setLike] = useState(0)
  const handleClick = () => {
    setLike(like + 1)
    setLike(like + 2)
    setLike(like + 3)
  }
  return (
    <div>
      <button onClick={handleClick}>{like} ✌</button>
    </div>
  );
}

点击按钮后，like会变成三，并不会是六

// setState
...
handleClick = () => {
  const {like} = this.state
  this.setState({like: like + 1})
  this.setState({like: like + 2})
  this.setState({like: like + 3})
}

render() {
  const {like} = this.state;
  return (
    <div>
    <button onClick={this.handleClick}>{like} ✌</button>
</div>
  );
}
...

同样，点击按钮后，state的值会是三，而不是六

  • 不同

如果使用单个 state 变量，每次更新 state 时需要合并之前的 state。因为 useState 返回的 setState 会替换原来的值。这一点和 Class 组件的 setState 不同。setState 会把更新的字段自动合并到 this.state 对象中。

// setState
this.state = {
  top: 0
  left: 0
}

如上，setState可以只针对某一个state值进行更新，更新完之后，React会自动合并对象

// useState
const [positions, setPositions] = useState({top: 0, left: 0})

如果state的值是一个对象，使用setPositions更改state时，如果只修改一个值如top时，未处理left，返回的positions将会被改变，导致错误。
