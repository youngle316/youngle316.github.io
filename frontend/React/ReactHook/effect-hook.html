# React-Hook 之 Effect Hook

Effect Hook 可以让你在函数组件中执行副作用操作。数据获取，设置订阅取消订阅及手动更新DOM都属于副作用。但是副作用又分为需要清除的和无需清除的。

# 使用 Effect Hook

useEffect可以看做是componentDidMount，componentDidUpdate 和 componentWillUnmount 这三个生命周期函数的组合。

在React的源码中，useEffect是这样定义的

function useEffect(effect: EffectCallback, deps?: DependencyList): void;

type EffectCallback = () => (void | (() => void | undefined));

type DependencyList = ReadonlyArray<any>;

# 无需清除的副作用

有时候，我们只想**在 React 更新 DOM 之后运行一些额外的代码。**比如发送网络请求，手动变更 DOM，记录日志，这些都是常见的无需清除的操作。

# useEffect的基本用法

通过DOM更新网页标题的例子来展示useEffect的基本用法。

  1. 使用class

只需要在上述对应的class代码中加上这两个生命周期函数即可

componentDidMount() {
  const {like} = this.state;
  document.title = `点击了${like}次`
}

componentDidUpdate() {
  const {like} = this.state;
  document.title = `点击了${like}次`
}
  1. 使用Hook
  • useEffect的简单使用
import React, {useState, useEffect} from 'react';

function LikeButtonHook(props) {
    const [like, setLike] = useState(0)
    useEffect(() => {
        document.title = `点击了${like}次`
    })
    return (
        <div>
            <button onClick={() => setLike(like +1)}>{like} ✌</button>
        </div>
    );
}

export default LikeButtonHook;

在上述默认的情况下，useEffect会在组件第一次加载和更新之后都会执行。后面会说明如何去控制它。

# 需要清除的副作用

# useEffect第一个参数effect的返回值

当设置了定时器或添加了事件监听时，我们需要在使用完之后及时的清除它，这样可以避免内存泄漏等不必要的问题。在class组件中通常是在componentWillUnmount中去执行卸载的操作。那么在useEffect中同样也需要及时的清除这个副作用，就用到了useEffect第一个参数effect的返回值，它是一个函数，如果return了一个函数，那么当清除副作用的时候会就会执行这个函数。下面用一个监听鼠标位置的例子来演示一下用法。

import React, {useState, useEffect} from 'react';

function MouseTrackerHook(props) {
  const [position, setPosition] = useState({x: 0 , y: 0})

  useEffect(() => {
    const handleClick = (e) => {
      console.log('inner')
      setPosition({x: e.clientX, y: e.clientY})
    }
    document.addEventListener('click', handleClick)
  })
  return (
    <p>
      X: {position.x} , Y: {position.y}
    </p>
  );
}

export default MouseTrackerHook;

上面的代码通过事件监听实现了一个监听鼠标点击位置的例子。但是却会有一个问题那就是点击页面后打印的inner不是线性增加的，点击了4次，显示inner打印了15次。原因就是没有及时的清除定时器。

将上面代码的useEffect部分进行修改

useEffect(() => {
  const handleClick = (e) => {
    console.log('inner')
    setPosition({x: e.clientX, y: e.clientY})
  }
  document.addEventListener('click', handleClick)
  return () => {
    document.removeEventListener('click', handleClick)
  }
})

这样修改了之后，inner就会是线性增加。

下面再添加一些日志来看看useEffect是怎么工作的

function MouseTrackerHook(props) {
  const [position, setPosition] = useState({x: 0 , y: 0})

  useEffect(() => {
    console.log('add effect', position.x)
    const handleClick = (e) => {
      console.log('inner', position.x)
      setPosition({x: e.clientX, y: e.clientY})
    }
    document.addEventListener('click', handleClick)
    return () => {
      console.log('remove effect', position.x)
      document.removeEventListener('click', handleClick)
    }
  })
  console.log('before render', position.x)
  return (
    <p>
      X: {position.x} , Y: {position.y}
    </p>
  );
}
  • 页面一加载的打印结果是
before render 0
before render 0
add effect 0
  • 点击一次的结果是
inner 0
before render 469
before render 469
remove effect 0
add effect 469

总结：

  1. 在参数effect中返回一个函数，是effect的可选机制。每个 effect 都可以返回一个清除函数。如此可以将添加和移除订阅的逻辑放在一起。它们都属于 effect 的一部分。
  2. React 会在组件卸载的时候执行清除操作，effect 在每次渲染的时候都会执行。React 会在执行当前 effect 之前对上一个 effect进行清除。

# 控制useEffect的触发

# useEffect的第二个参数

useEffect的第二个参数deps是一个只读的数组。它使用的方式分为以下三种情况。

  1. 不传第二个参数

当不传第二个参数时，组件在挂载，更新，卸载时都会触发useEffect

  1. 传入空数组

组件只会在挂载和卸载的时候才会触发useEffect

  1. 传入一个state为元素的数组

只有当数组中的任意一个state的值改变了才会触发useEffect
